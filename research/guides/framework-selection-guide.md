# Framework Selection Guide

A practical guide for choosing which S-P-F-H framework to use in different situations.

---

## Quick Decision Matrix

| Your Situation | Recommended Framework | Why |
|---|---|---|
| Well-defined coding task, need to ship | **None** — use standard prompt engineering | S-P-F-H frameworks add overhead that doesn't help routine tasks |
| Creative project, open-ended exploration | **Collaborative Intelligence Patterns** | Maximizes emergent, unexpected outcomes through partnership dynamics |
| Tight deadline, limited budget or resources | **Resource-Constrained Innovation** | Reframes limitations as design parameters rather than obstacles |
| Extended AI session, feeling mentally foggy | **Consciousness-Driven Development** | Awareness practices restore evaluative engagement |
| Complex project spanning multiple concerns | **Meta-Framework Integration** | Provides structured approach to combining multiple perspectives |
| New to S-P-F-H | **Consciousness-Driven Development** | The PAUSE → INQUIRE → EXPLORE → SYNTHESIZE → IMPLEMENT cycle is the most immediately practical entry point |

## Detailed Selection Criteria

### Axis 1: Task Type

```
Well-defined, routine ──────────────────── Creative, exploratory
    Standard prompting                      Collaborative Intelligence
    is sufficient                           Patterns add most value
```

Research finding: Human-AI collaboration benefits are strongest for creative and content-generation tasks and weakest for analytical/decision-making tasks (Gaissmaier et al., 2024). The S-P-F-H frameworks add the most value where standard prompt engineering leaves the most gap — in open-ended, creative, and ambiguous work.

### Axis 2: Resource Availability

```
Resources abundant ──────────────────── Resources constrained
    Collaborative Intelligence             Resource-Constrained
    or Consciousness-Driven                Innovation
```

When resources are abundant, the choice is between depth of collaboration (Collaborative Intelligence) and quality of individual practice (Consciousness-Driven). When resources are constrained, the Resource-Constrained Innovation framework directly addresses how to use limitations productively.

**Important caveat:** If resources are so constrained that basic cognitive function is impaired (severe sleep deprivation, extreme time pressure, financial crisis), the frameworks may add cognitive load rather than reducing it. Stabilize first, then apply frameworks. See Foundation Paper 2 on the scarcity bandwidth tax (Mullainathan & Shafir, 2013).

### Axis 3: Individual vs. Team

```
Solo practitioner ──────────────────── Team/network
    Consciousness-Driven                Collaborative Intelligence
    Development                         Patterns (network scaling)
```

Solo practitioners benefit most from Consciousness-Driven Development's personal practice protocols (awareness check-ins, PAUSE cycle, reflective practice). Teams benefit from Collaborative Intelligence Patterns' emphasis on network intelligence, shared knowledge commons, and distributed problem-solving.

### Axis 4: Familiarity with Frameworks

```
New to S-P-F-H ──────────────────── Experienced practitioner
    Start with CDD's                  Meta-Framework Integration
    PAUSE cycle                       for cross-framework synthesis
```

New practitioners should start with one framework (Consciousness-Driven Development is most accessible) and master its core practices before combining frameworks. Meta-Framework Integration is designed for practitioners who already understand the individual frameworks.

## When NOT to Use Each Framework

### Collaborative Intelligence Patterns
**Skip when:**
- The task is well-defined and you already know the solution approach
- You need speed more than depth
- You're doing routine maintenance or bug fixes
- The "partnership" framing feels forced or unnatural for the task

### Consciousness-Driven Development
**Skip when:**
- You're in a productive flow state — don't interrupt it with awareness check-ins
- The task is purely mechanical (data formatting, boilerplate generation)
- Time pressure is at the threshold of impossibility (stabilize first)
- The contemplative vocabulary creates more resistance than engagement

### Resource-Constrained Innovation
**Skip when:**
- Resources are genuinely abundant and the constraint framing would be artificial
- The constraint is one you should remove rather than embrace (a solvable organizational problem, not a creative parameter)
- The constraint is causing cognitive impairment rather than creative focus

### Meta-Framework Integration
**Skip when:**
- You haven't used at least two individual frameworks in practice
- The task fits cleanly within a single framework's scope
- You're tempted to add meta-framework complexity to a simple problem
- The recursive language feels circular rather than clarifying

## Common Combinations

### Creative Project Under Time Pressure
**Resource-Constrained Innovation + Consciousness-Driven Development**

Use RCI to reframe the time constraint as a focusing function ("what's the essential 20%?"). Use CDD's PAUSE cycle at transition points to maintain judgment quality under pressure.

### Team Brainstorming Session
**Collaborative Intelligence Patterns + Consciousness-Driven Development**

Use CIP's session initiation protocol (ALIGN → ATTUNE → INTEND → COMMIT) to establish the collaborative dynamic. Use CDD's awareness practices when the team notices energy or attention flagging.

### Long-Term Research or Exploration
**All four frameworks through Meta-Framework Integration**

For extended projects that span creative exploration, resource management, personal practice, and collaborative dynamics, use the meta-framework to select which individual framework is most appropriate for each phase.

## The 30-Second Version

1. **Routine task?** Skip the frameworks, use standard prompt engineering.
2. **Creative task?** Collaborative Intelligence Patterns.
3. **Constrained?** Resource-Constrained Innovation.
4. **Feeling foggy or disengaged?** Consciousness-Driven Development's PAUSE cycle.
5. **Everything at once?** Meta-Framework Integration.
6. **Not sure?** Start with Consciousness-Driven Development — it's the most portable.
