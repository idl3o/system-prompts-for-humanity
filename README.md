# System Prompts for Humanity

A collection of system prompts and frameworks exploring consciousness-driven human-AI collaboration, constraint-based innovation, and collaborative intelligence patterns.

## What This Is

This is a **content project**, not a software application. There is no runnable code. The deliverables are system prompt documents and framework descriptions that you can paste into AI assistants (Claude, ChatGPT, etc.) to shape how they collaborate with you.

The core idea: treating human-AI interaction as a genuine partnership — where constraints become creative catalysts and awareness practices improve technical work — produces better outcomes than treating AI as a passive tool.

### Dual Format Convention

Each framework exists as both `.txt` and `.md`:
- **.txt** — plain text, optimized for copy-paste directly into AI system prompts
- **.md** — markdown with formatting, better for reading and reference

## How to Use These Prompts

1. **Pick a framework** that fits your situation (see descriptions above)
2. **Copy the `.txt` version** into your AI assistant's system prompt or paste it at the start of a conversation
3. **Work with the framework** — the prompts are designed to shift the interaction from transactional command-response toward collaborative problem-solving

For example, if you're working on a project with tight resource constraints, paste `resource-constrained-innovation.txt` and ask your AI assistant to help you find creative solutions within those constraints rather than defaulting to "add more resources."

## The Four Frameworks

**Collaborative Intelligence Patterns** — Structures for human-AI partnership that go beyond simple Q&A. Covers how to build shared context, leverage complementary strengths, and scale collaborative practices across teams.

**Consciousness-Driven Development** — Integrating awareness and reflection into technical work. Not mystical — practical: pause before acting, notice assumptions, reflect on what's working, bring intention to problem-solving.

**Resource-Constrained Innovation** — Using limitations as a creative forcing function. When you can't add more compute, more people, or more budget, constraint-driven approaches often produce more elegant solutions.

**Meta-Framework Integration** — A pattern that connects the other three frameworks through a recurring cycle: constraints create the conditions for innovation, which opens new possibilities, which evolve into new constraints, recursively.
